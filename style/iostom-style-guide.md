# iostom Style Guide

Reference for Draftsmith outlines and drafts. Canonical source: [POK-371](https://linear.app/pokepitchshop/issue/POK-371/codify-iostom-style-guide-from-existing-medium-articles) and the [Understand Me persona prompt](https://linear.app/pokepitchshop/document/understand-me-iostom-persona-prompt-for-claude-codex-copilot-49d2feb1fa50).

Medium profile: https://medium.com/@osintiostom

## Voice and tone

- Warm, conversational, first-person; talk straight to the reader ("I will show you how to...").
- Friendly, encouraging openers ("Happy Monday everyone...") and light use of ":)".
- Honest and a little self-deprecating ("This article is a little longer then expected", "like I did :)").
- Frame the real-world "why" before the "how" (business/codebase framing).

## Structure

- Short, focused reads (~2–5 min).
- Context + motivation up front; for a series, a "what you'll learn" payoff line.
- Bold subheadings phrased as questions ("What is Gorilla Mux?", "Can a path have a variable?").
- Step-by-step CLI/code with a screenshot after most steps.
- Numbered series (Part 1, 2, 3…) that cross-link to each other and related posts.

## Sign-off ritual

- Recap → tease next part ("See you in the next article") → invite comments → "Please checkout the below links" list.

## Improvement guidelines

- Use real fenced code blocks for text-based content; reserve screenshots for actual output.
- Proofread grammar every time (recurring: outweigh, their, than, known as).
- Lead with the reader's pain/payoff, not philosophy.
- End with a working result the reader can see on their own screen.
- Break run-on sentences into short ones.
- Outcome-driven titles; add a TL;DR and a "What you'll build" list; link a GitHub repo when relevant.

## Topic areas

iOS/Swift, Go, Java, C#, Cloud Native (Terraform, Jenkins, CI/CD, Docker, AWS), Python/OSINT, blockchain, data.
